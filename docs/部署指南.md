# DrawAgent 部署指南

## 1. 系统要求

| 项目 | 最低 | 推荐 |
|------|------|------|
| OS | Ubuntu 20.04+ / CentOS 7+ | Ubuntu 22.04 LTS |
| Python | 3.11+ | 3.11 |
| Node.js | 20+ | 20 LTS |
| GPU | 可选 | NVIDIA A100/A800（MinerU VLM 推理） |
| VRAM | 8 GB+ | 40 GB+（并行解析） |
| RAM | 16 GB | 32 GB+ |
| Disk | 10 GB | 50 GB+（含模型缓存） |

无 GPU 时可跳过 MinerU 解析，系统自动降级为 mock 模式。

## 2. 安装步骤

**Step 1 — 克隆仓库**

```bash
git clone https://github.com/your-team/drawagent.git
cd drawagent
```

**Step 2 — 创建虚拟环境**

```bash
python3.11 -m venv .venv
source .venv/bin/activate
```

**Step 3 — 安装依赖**

```bash
pip install -r requirements.txt
pip install mineru          # GPU 本地解析时安装
```

`requirements.txt` 含 FastAPI、uvicorn、PyMuPDF、pdf2image、OpenCV、pandas 等。
FAISS / BGE-M3 检索组件为可选项，按需取消注释。

**Step 4 — 配置环境**

项目根目录创建 `.env`（已 gitignore）：

```
LLM_API_KEY=your-api-key
LLM_BASE_URL=https://api.deepseek.com
LLM_MODEL=deepseek-chat
```

`LLM_BASE_URL` / `LLM_MODEL` 采用 OpenAI 兼容协议，可切换任意厂商。

**Step 5 — MinerU 解析图纸（首次）**

将 PDF 放入 `data/origin/`，然后执行：

```bash
PYTHONUNBUFFERED=1 MINERU_MODEL_SOURCE=modelscope \
  python -m src.adapters.mineru_real data/origin/ \
  -b hybrid-auto-engine -m auto -l ch -w 4
```

`-w 4` 表示 4 路并行，VRAM 不足时改为 `-w 1`。结果写入 `data/mineru/`，
系统检测到该目录后自动禁用 mock 模式。

**Step 6 — 前端依赖**

```bash
cd frontend && npm install && cd ..
```

## 3. 启动系统

```bash
# 后端（默认 127.0.0.1:8000）
source .venv/bin/activate
python server.py

# 前端（默认 localhost:5173，/api 自动代理到后端）
cd frontend && npm run dev
```

浏览器打开 **http://localhost:5173** 即可使用。

## 4. 验证

```bash
# 图纸接口
curl http://127.0.0.1:8000/api/drawings

# 对话接口（SSE 流式）
curl -N -X POST http://127.0.0.1:8000/api/chat \
  -H "Content-Type: application/json" \
  -d '{"prompt": "设计号是什么？"}'
```

正常时图纸接口返回 JSON manifest；对话接口返回 reasoning / content / tool_start / tool_result / done 等 SSE 事件。

## 6. 配置参考

所有变量可在 `.env` 设置或通过系统环境变量覆盖：

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `HOST` | 服务绑定地址 | `127.0.0.1` |
| `PORT` | 服务端口 | `8000` |
| `RENDER_DPI` | PDF 渲染 DPI | `300` |
| `MINERU_ROOT` | MinerU 结果目录 | `data/mineru` |
| `USE_MOCK_MINERU` | mock 解析器（0/1） | 自动检测 |
| `LLM_API_KEY` | LLM API 密钥 | 空 |
| `LLM_BASE_URL` | LLM API 地址 | `https://api.deepseek.com` |
| `LLM_MODEL` | LLM 模型名称 | `deepseek-chat` |

`USE_MOCK_MINERU` 默认根据 `MINERU_ROOT` 是否存在自动判断。

## 7. 常见问题

**"未加载图纸"** — 确认 `data/origin/` 含 PDF，`data/mineru/` 有解析输出，`MINERU_ROOT` 指向正确。

**"后端未连接"** — 确认 `server.py` 在运行，端口 8000 未被占用，`frontend/vite.config.ts` 中 proxy 地址匹配。

**MinerU OOM** — 将 `-w 4` 改为 `-w 1` 串行解析，或降低 `gpu_memory_utilization`。

**HuggingFace 下载失败** — 设置 `MINERU_MODEL_SOURCE=modelscope` 使用国内镜像。

**pdf2image 报错 "poppler not found"** — `sudo apt-get install poppler-utils`（Ubuntu）或 `sudo yum install poppler-utils`（CentOS）。
