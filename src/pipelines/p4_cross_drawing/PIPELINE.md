## 触发
- "建筑和结构的设计号是否一致"
- "核对所有图纸设计号是否一致"
- "核对图纸目录所列图纸是否齐全"
- "有没有图号冲突 / 缺失的图纸"

## 流程
1. 逐图提取图签字段（设计号、图号），识别目录页并解析目录条目
2. 四类确定性规则检查（零 LLM，结果可复现）：
   - 设计号一致性：全部图纸的设计号是否统一，不一致时按专业列出分布
   - 目录缺图：目录所列图号在图纸包中是否有对应图签
   - 图号冲突：同专业同图号是否被多张图纸占用
   - 引用完整性：跨图引用与国标图集引用的提取与统计
3. 规则结论同步写入 `TaskResult.data`（design_nos / design_no_consistent /
   missing / conflicts / atlas_refs / ref_total / checked_drawings），
   与 markdown 报告一一对应，供导出接口与测试直接消费
4. 修复建议层（可选 LLM）：配置 `LLM_API_KEY` 后，把规则检查报告交给
   LLM（施工图审查顾问角色），生成按优先级排序的修复建议，说明每项的
   影响面与处理顺序依据；未配置时该小节以一行说明跳过，
   规则检查结果不受任何影响
