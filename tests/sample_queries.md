# DrawAgent 示例问题清单

> 涵盖 5 个 Pipeline 的典型触发场景，可根据本地图纸名称替换目标图纸。

## 一、单图问答（P1 drawing_qa）

| # | 问题 | 目标图纸 | 预期路由 | 期望返回 |
|---|------|----------|----------|----------|
| 1 | 这张图纸的设计号是什么？专业负责人是谁？ | 结构/结构设计说明 | p1_drawing_qa | 图签字段 |
| 2 | 这张图的图号和版本号？ | （任选） | p1_drawing_qa | 图签字段 |
| 3 | 建筑设计说明里的层高是多少？ | 建筑/建筑设计说明 | p1_drawing_qa | 层高数值 |

## 二、表格提取（P2 table_extract）

| # | 问题 | 目标图纸 | 预期路由 | 期望返回 |
|---|------|----------|----------|----------|
| 4 | 提取门窗统计表中所有表格的数据 | 建筑/门窗统计表 | p2_table_extract | 结构化表格记录 |

## 三、批量解析统计（P3 batch_parse）

| # | 问题 | 预期路由 | 期望返回 |

## 四、跨图比对（P4 cross_drawing）

| # | 问题 | 预期路由 | 期望返回 |

## 五、解析质量验证（P5 quality_verify）

| # | 问题 | 预期路由 | 期望返回 |

| # | 问题 | 自动分解 | 期望 |
|---|------|----------|------|

## 复现方式

**方式 A：一键脚本**

```bash
python tests/run_demo.py
```

会按上述清单顺序跑示例，并在本地生成运行日志。

**方式 B：前端交互**

1. 启动后端：`python server.py`
2. 启动前端：`cd frontend && npm run dev`
3. 浏览器打开 `http://localhost:5173`
4. 点击对话栏中的示例按钮，或直接输入上表问题

**方式 C：API 直调**

```bash
curl -N -X POST http://127.0.0.1:8000/api/chat \
  -H "Content-Type: application/json" \
  -d '{"prompt": "检查所有图纸的设计号是否一致"}'
```
